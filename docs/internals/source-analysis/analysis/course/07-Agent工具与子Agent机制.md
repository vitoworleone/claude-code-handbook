# 第七集：Agent 工具与子 Agent 机制

大家好，欢迎来到第七课——Agent 工具与子 Agent 的机制。

Claude Code 里有一个工具叫 `AgentTool`，它的 description 是这样的：

> 启动一个新的 Claude 实例来执行子任务。可以并行运行多个 Agent，在后台执行长时间任务。

注意关键词：**「启动一个新的 Claude 实例」**。这意味着这不是普通的工具调用——这是一个 Claude 调用另一个 Claude。

这集我们来看这个「元工具」是怎么实现的，以及它为什么有四种不同的执行模式。

## 核心价值

### 并行化

想象你让 Claude「帮我并行分析项目里所有 Python 文件的代码质量」。对于 1000 个文件，单线程逐一分析太慢了。

用 `AgentTool`：主 Agent 可以创建多个子 Agent，每个处理一批文件。把 1000 个文件分成 10 批，每批 100 个——10 个子 Agent 同时工作，速度提升 10 倍。

### 隔离

把危险操作放到子 Agent 里执行，失败了也不影响主对话。这在处理敏感操作时特别有用。

## 四种执行模式

这四种模式代表了在「并行化」和「隔离」这两个维度上的不同选择：

| 模式 | 参数 | 特点 | 适用场景 |
|------|------|------|----------|
| **本地同步** | 默认 | 等子 Agent 完成才继续 | 快速子任务（≤15 秒） |
| **本地后台** | `runInBackground: true` | 立即返回，后台执行 | 耗时任务的并行化 |
| **Fork Agent** | `isolation: 'worktree'` | 文件系统级别隔离 | 需自由修改但不污染主目录 |
| **Remote Agent** | `isolation: 'remote'` | 云端 CCR 运行 | 小时级长任务或特定环境 |

从本地同步到 Remote Agent，隔离程度越来越高，适用场景也越来越复杂。

## 本地同步模式（默认）

主 Agent 调用 `AgentTool`，等待子 Agent 完成，拿到返回结果后继续执行。很简单，适合 15 秒以内的快速任务。

## 本地后台模式

关键看流程：

1. 主 Agent 调用 `AgentTool` → 立即得到响应：「任务已在后台启动」，返回一个 Agent ID。
2. 子 Agent 在后台执行任务，输出写入 `~/.claude/tasks/` 目录下的日志文件。
3. 主 Agent 拿到 Agent ID 后继续做其他事情。
4. 稍后用 `TaskOutput` 工具检查进度，或直接读取输出文件。

适合代码重构、文档生成、完整测试套件运行等耗时任务。

### 15 秒自动后台化

在「驻守模式」（通过 WhatsApp 或 Telegram 等消息应用发送任务的场景）里有一个特殊机制：

1. 用户发送任务后，主 Agent 调用 `AgentTool` 开始同步执行。
2. 系统监控执行时间。
3. 一旦超过 15 秒，系统检测到任务还在运行且当前是驻守模式。
4. 自动调用 `backgroundCurrentAgent` 把任务转到后台。
5. 告诉用户「任务已在后台执行，完成时会通知你」。

**为什么是 15 秒？** 因为在消息应用里，用户发送任务后通常等 2-3 秒没结果就去做别的事了。15 秒是一个平衡点——比用户预期同步等待的上限稍长，但不会让用户等太久。

这个设计体现了一个深刻的洞察：**异步是正确的默认行为，同步才是特殊需要。** 这很反直觉，但确实是这样设计的。

## Fork Agent（Worktree 模式）

当主 Agent 调用 `AgentTool` 并设置 `isolation: 'worktree'` 时，系统做三件事：

**1. 创建 Git Worktree**

这是 Git 的高级功能，允许同一个仓库有多个工作目录，每个工作目录可以在不同分支上。

**2. Fork 文件状态缓存**

文件状态缓存记录了上次读取时某个文件的内容。如果子 Agent 修改了文件并更新了缓存，主 Agent 下次读缓存就会得到错误的快照。所以要 fork 一份快照——子 Agent 修改自己的快照，不影响主 Agent。

**3. 复制系统提示**

这里有一个优化：Claude API 有 Prompt Cache 机制。相同的系统提示命中缓存可以大幅降低 Token 成本。所以子 Agent 会复用主 Agent 的系统提示，确保缓存命中。

任务完成后直接丢弃 Worktree——主目录干干净净。

**代价也有三个：**
- 项目必须是 Git 仓库
- Worktree 创建有约 100ms 的额外开销
- 修改在独立分支，需要合并才能应用到主分支

## Remote Agent

使用 `isolation: 'remote'` 时，子 Agent 在云端通过 Claude Code Runner (CCR) 运行：

1. 主 Agent 检查 CCR 是否可用。
2. 上传必要文件到云端。
3. 子 Agent 在云端拥有独立计算资源。
4. 执行完成后，通过返回的 Session ID 查看结果。

**特点：完全隔离，不占用本地资源，环境完全独立。** 适合小时级别的长任务或需要特定操作系统的任务。
