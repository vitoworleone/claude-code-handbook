# 第4章 会话管理实战

Claude Code 的会话会自动保存，你可以随时中断、恢复、分支或清空。但会话不是越多越好——管理不当会导致"找不到之前的对话""上下文混乱""Claude 变蠢"等问题。本章讲怎么组织会话，让它们成为资产而非负担。

---

## 4.1 会话生命周期与数据流向

一个会话从创建到消失，经历以下阶段：

```
创建 → 活跃 → 压缩 → 清空 → 恢复 → 归档 → 删除
```

### 数据存在哪

会话记录以 JSONL 格式保存在本地：

```
~/.claude/projects/<project-name>/<session-id>.jsonl
```

每一行是一个 JSON 对象，记录消息、工具调用和元数据。这些文件：
- **默认保留 30 天**，之后自动清理（可通过 `cleanupPeriodDays` 设置调整）
- 完全离线，不上传云端
- 可通过 `CLAUDE_CONFIG_DIR` 环境变量改存放位置

### 导出会话

```bash
/export                    # 复制到剪贴板
/export my-session.txt     # 保存为文本文件
```

导出内容是纯文本格式，方便存档或分享。

### 彻底禁用持久化

```bash
claude -p --no-session-persistence "query"
# 或设置环境变量
export CLAUDE_CODE_SKIP_PROMPT_HISTORY=1
```

适合处理敏感数据或一次性任务。

## 4.2 命名与组织策略

### 什么时候命名

| 时机 | 方式 | 场景 |
|------|------|------|
| 启动时就知道要干嘛 | `claude -n "feature-auth"` | 有计划的任务 |
| 做到一半才明确方向 | `/rename "fix-race-condition"` | 探索中发现重点 |
| 从 Plan Mode 批准 | 自动命名 | 基于 plan 内容生成 |

### 命名约定建议

好的命名让你一周后还能找到：

```
feature/user-auth        # 功能开发
pr-123-review            # 审查某个 PR
bug/flaky-test-2026-05   # 修复问题 + 日期
refactor/api-types       # 重构 + 范围
explore/mcp-integration  # 探索性任务（可能无结果）
```

**不要命名的情况**：
- 一次性探索（"看看这个文件是干嘛的"）
- 5 分钟能搞定的小事
- 明确用完即走的查询

### 命名与恢复

命名后会话可以通过名字直接恢复：

```bash
claude --resume "feature-auth"
```

名字在**当前仓库及其 worktree** 范围内解析。如果多个会话同名，`--resume` 会打开选择器让你挑。

## 4.3 恢复策略

### 三种恢复方式

| 方式 | 命令 | 场景 |
|------|------|------|
| 继续最近 | `claude --continue` | 最常见的恢复，接着刚才的干 |
| 指定恢复 | `claude --resume "name"` 或 `claude --resume <id>` | 切换到另一个任务 |
| 从 PR 恢复 | `claude --from-pr 123` | PR 被退回，回到修改现场 |

### 会话选择器技巧

运行 `/resume` 或 `claude --resume`（不带参数）打开交互式选择器：

| 快捷键 | 功能 |
|--------|------|
| `↑`/`↓` | 导航 |
| `Enter` | 恢复选中 |
| `Space` | 预览内容 |
| `Ctrl+R` | 重命名 |
| `/` | 搜索过滤 |
| `Ctrl+A` | **所有项目**的会话 |
| `Ctrl+W` | 当前仓库**所有 worktree** 的会话 |
| `Ctrl+B` | 仅当前 git 分支的会话 |

**跨项目恢复**：按 `Ctrl+A` 后，选中其他项目的会话不会直接恢复，而是把 `cd` + `claude --resume` 命令复制到剪贴板，你手动执行即可。

### 后台会话恢复

后台会话（`--bg` 启动的）标记为 `bg`，也出现在选择器中：

```bash
claude attach <session-id>    #  attach 到后台会话
claude logs <session-id>      #  查看后台会话输出
claude respawn <session-id>   #  重启后台会话
```

## 4.4 分支与隔离

### 什么时候分支

分支创建当前会话的副本，让你尝试不同方案而不影响主线：

- **A/B 测试**："试试方案 A，不行再回方案 B"
- **安全实验**："重构这个模块，但怕搞坏"
- **并行审查**：一个分支实现，一个分支审查

```bash
# 会话内分支
/branch try-streaming-approach

# 从命令行分支（恢复时创建分支）
claude --continue --fork-session
```

分支后会打印两个 session ID：新分支和原会话。原会话保持不变。

**注意**：分支后权限不继承——原会话中"允许本次"的权限在新分支中需要重新确认。这不是 bug，而是**故意的安全设计**。论文对这一设计决策的解释是："sessions are treated as isolated trust domains"——每个会话被视为独立的信任域。恢复之前授予的权限会在一个已经变化的上下文中携带过时的信任决策。架构选择了"重新授予"而非"隐式持久化"，接受用户摩擦作为维护安全不变式的代价。

### Worktree 隔离

Worktree 是文件系统级别的隔离，比分支更重：

```bash
# 创建隔离 worktree
claude --worktree feature-auth

# 从 PR 创建
claude --worktree "#1234"

# 自动生成名称
claude --worktree
```

Worktree 创建在 `.claude/worktrees/<name>/`，分支名为 `worktree-<name>`。完成且无更改时自动删除。

| | 分支 (`/branch`) | Worktree (`--worktree`) |
|--|------------------|------------------------|
| 隔离级别 | 会话/上下文 | 文件系统 + git 分支 |
| 用途 | 尝试不同对话路径 | 并行开发不同功能 |
| 资源开销 | 低 | 中等（额外文件系统） |
| 自动清理 | 否 | 是（无更改时） |

## 4.5 压缩与清空策略

### `/clear` vs `/compact`

| 命令 | 效果 | 什么时候用 |
|------|------|-----------|
| `/clear` | 清空上下文，开始新对话。旧对话保存可恢复 | 话题切换、上下文跑偏、同一问题纠正多次 |
| `/compact [instructions]` | 把历史摘要化，腾出空间 | 上下文快满了但还要继续当前话题 |

### 黄金法则

**同一问题纠正 2 次以上 → `/clear` 并用更好的提示重新开始。**

不要在一个已经跑偏的上下文里死磕。Claude 会受之前错误尝试的影响，越纠越偏。

### 上下文满了的信号

- Claude 开始重复自己已经说过的话
- 问已经回答过的问题
- 建议明显变蠢（忘记你刚给的约束）
- 响应变慢（注意力机制 O(n²)，token 越多越慢）

### 查看上下文消耗

```bash
/context
```

显示当前上下文的使用情况，帮助你判断是否需要压缩。

### 自动压缩设置

在 `settings.json` 中开启自动压缩：

```json
{
  "autoCompact": true,
  "autoCompactThreshold": 0.8
}
```

当上下文达到阈值时自动触发压缩。

### 压缩对配置的影响

- **CLAUDE.md**、skills、hooks、规则文件：压缩不影响，始终保留
- **对话历史**：被摘要化，具体 tool 输出可能丢失
- **内存文件**：不受影响

## 4.6 会话健康检查

### 诊断清单

当你感觉"这个会话不太对劲"时，按以下顺序检查：

| 检查项 | 命令 | 正常标准 |
|--------|------|---------|
| 上下文有多满？ | `/context` | 低于 80% |
| 历史是不是太长？ | 看消息数量 | 超过 20 轮考虑压缩 |
| 有没有互相矛盾的前置指令？ | 回顾之前的消息 | 同一约束只出现一次 |
| CLAUDE.md 是否加载？ | 启动时的提示 | 应该显示 "Loaded CLAUDE.md" |

### 处置决策树

```
会话不对劲？
  ├── 上下文 > 80%？
  │     ├── 还要继续当前话题？ → /compact
  │     └── 要换话题？ → /clear
  ├── 同一问题纠正 2 次+？
  │     └── → /clear，用更好的提示重来
  ├── Claude 忘记之前的约束？
  │     └── → /clear 或把约束写进 CLAUDE.md
  └── 其他奇怪行为？
        └── → /clear 是最安全的重置方式
```

### 预防性维护

- **每完成一个子任务**：考虑 `/compact` 释放空间
- **每天结束工作**：给会话起个好名字（`/rename`），方便明天恢复
- **每周清理**：用 `claude project purge` 删除不再需要的老会话
